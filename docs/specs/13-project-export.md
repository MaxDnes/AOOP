## Per-tool export buttons (same agents as spec 14, see ownership there)
Each tool adds an "Export project (.zip)" button beside its copy buttons,
with a project-name input (sensible default: designer = window Title
sanitized, testlab = "ExamApp.Tests", querylab = "QueryConsole", asynclab =
"AsyncApp"). Wire to window.PROJZIP (the core's global):
- Designer: avaloniaProject from current design (AXAML + VM + Models pane
  files + reference image NOT included). Both exam presets must export to
  projects that `dotnet build` green.
- Test Lab: xunitProject from current pasted sources + generated tests +
  headless flag from the mode toggles.
- Query Lab: consoleProject from generated Program.cs + the pasted sample
  JSON saved as the data file name used in the code.
- Async Composer: avaloniaProject from generated VM + its AXAML snippet
  wrapped in the scaffold's MainWindow.
Each tool also keeps a graceful path when window.PROJZIP is missing (button
hidden) so tools never crash if load order changes.

## Definition of done

Tool agents: export produces a zip whose expanded project builds green
(verify with Node-generated bytes + Expand-Archive + dotnet build for YOUR
tool's default/preset content; report build output). Full suite green at
session end.
